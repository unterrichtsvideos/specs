## Zusammenfassung Das Dokument stellt ein **Anwendungsprofil** für XML-Feeds im Kontext des Meta-Videoportal [unterrichtsvideos.net](https://unterrichtsvideos.net/metaportal/) dar und ist angelehnt an die Spezifikationen von [[RSS 2.0]] und [[XML 1.1]] zur Strukturierung des Datenaustauschs. Der Feed dient dem Austausch von Metadaten zwischen den Portalen/Anbietern (wie beispielsweise den Videoportal [ProVision](https://www.uni-muenster.de/ProVision/)) und dem Meta-Videoportal unterrichtsvideos.net.

## Tabelle / Schema des `use_restrictions`-Felds – Erklärung und Beispiel Das `use_restrictions`-Feld definiert die Zugriffsmöglichkeiten für ein bestimmtes Video-Item und bestimmte vordefinierte "Nutzergruppen", die in [[[#useRestrictionsTabelle]]] aufgeführt werden (G0 – G7). Für jede dieser Gruppen kann nach dem Doppelpunkt ein eigenes Zugriffslevel festgelegt werden. "Nutzergruppen" ohne Zugriff (Zugriffslevel = 0) __können__ als Werte weggelassen werden. ```html "example": "Verwendung" use_restrictions = G1:1 | G2:2 | G6:3 ```

### Definition des Zugriffslevels
Wert Zugriffslevel Beschreibung Effekt auf Metaportal-Filter
0 Kein Zugriff (selbsterklärend) Video wird nicht angezeigt, wenn diese Nutzergruppe ausgewählt ist.
1 Lokaler Zugriff

Auf der Grundlage des Standorts des RSS-Anbieters und der angegebenen Nutzergruppe werden entweder die Universität oder das Bundesland, das mit dem Channel/Provider verbunden ist, automatisch zugewiesen.

Beispiel: ProVision wird von der WWU Münster bereitgestellt: Abhängig von einer bestimmten Nutzergruppe, die Level 1 (lokaler Zugriff) auf das Video hat, wird das zugehörige Bundesland = NRW oder die Universität = WWU Münster als Zugriffsmöglichkeit auf dieses spezifische Videoelement abgebildet.

Das Video wird angezeigt, wenn der ausgewählte Standort mit dem Standort der Universität (Provider) oder dem zugehörigen Bundesland (je nach Nutzergruppe) identisch ist.

Es wird nur der Standort eines bestimmten RSS-Providers / Videoportals / Channels abgebildet.
2 Trust network Zugriff Alle Universitäten und Bundesländer, die zum Metaportal beitragen und Partner des Trust Network sind (Kooperationsvertrag). Das Video wird angezeigt, wenn eine Kooperationsuniversität oder das zugehörige Bundesland (je nach Benutzergruppe) im Filter ausgewählt wurde.
3 Globaler Zugriff Zugriffsmöglichkeit für einen breiten Nutzerkreis (z. B. alle verifizierten legitimierten SSO-Nutzer). Dieses Level führt dazu, dass jede Universität oder jedes Bundesland im Metaportal-Filter abgebildet wird. Das Video wird auf Metaportal angezeigt, unabhängig davon, welche Nutzergruppe und welchen Ort der Benutzer auf der Suchseite als Filter auswählt.
### Beispiel

Feld: `use_restrictions` = G1:1 | G2:2 | G6:3
Hinweis: Nutzergruppen ohne Zugriff (Zugriffsstufe = 0) können weggelassen werden.

Bedeutung des Wertes `use_restrictions` im Zusammenhang mit ProVision als Provider des RSS-Feeds.

G1:1 bildet lokale Zugriffsmöglichkeit der Gruppe "Lehramtsstudierende" ab = Nur Studierende der "WWU Münster" (Channel Provider) können das Video sehen

G2:2 bildet die trust network Zugriffsmöglichkeit der Gruppe "Lehrkräfte im Vorbereitungsdienst" ab = Alle Gruppenmitglieder, die sich in einem Bundesland befinden, das mit dem Standort der zum trust network beitragenden Universitäten verbunden ist (Kooperationsvertrag), können das Video sehen.

G6:3 bildet die globale Zugriffsmöglichkeit der Gruppe "Lehrkräfte" ab = Alle Gruppenmitglieder in allen Bundesländern und Universitäten können das Video sehen.

Denken Sie daran: Sie können das Zugriffslevel für jedes einzelne Video festlegen, aber bedenken Sie, dass es sich hierbei nur um eine Zuordnung der beschreibenden Nutzungsbeschränkung handelt (natürlich gibt es keinen Einfluss oder Nebeneffekte in Bezug auf die tatsächlichen Zugriffsmöglichkeiten auf Ihre Videos, wenn sie falsch angegeben werden). Der Wert wirkt sich nur auf die Filteroptionen im Metaportal aus. Wenn Sie sich unsicher sind, folgen Sie der Prämisse: Lieber das Video anzeigen lassen, als es ausfiltern. Verwenden Sie das Feld `use_restrictions_label`, um den Benutzern mitzuteilen, wie sie möglicherweise auf das Video zugreifen können.
